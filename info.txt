檔案 / 資料夾,比喻,核心作用
venv/,獨立實驗室,存放所有需要的套件，不影響電腦其他地方。
app/ (資料夾),餐廳主體,存放網站核心邏輯的所有程式碼。
app/__init__.py,總機 / 開店準備,初始化 Flask，將 app 資料夾變成一個可以被匯入的「包」。
app/routes.py,菜單與服務生,定義網址（URL）與處理邏輯（函式）的對應關係。
microblog.py,啟動開關 / 正門,專案的最外層入口，告訴伺服器如何啟動這間餐廳。

深入檔案細節
1. microblog.py (最外層入口)
這是你給伺服器看的「啟動說明書」。當你執行 flask run 時，它會來到這裡找那個 app 實體。

內容通常只有一行： from app import app

作用： 它是連接「內層核心程式碼」與「外層執行環境」的橋樑。

2. app/ 資料夾 (核心包)
在 Python 中，只要資料夾內包含 __init__.py，這個資料夾就會變成一個 Package (包)。這讓你可以在其他地方寫 from app import ...。

3. app/__init__.py (初始化)
它是這間店的負責人。

作用： 1. 建立 Flask 物件實體。 2. 設定專案的基本配置（例如資料庫連線、金鑰）。 3. 最重要的一步：在最後一行導入 routes，把「菜單」貼到店門口。

4. app/routes.py (路由與邏輯)
這是你寫最多程式碼的地方。

作用： 當客人造訪 /login 或 /profile 時，這份檔案會告訴 Python 該去執行哪一段程式碼，並回傳什麼網頁內容給客人。

###為什麼有時要使用WriteOnlyMapped###
特性　　,　so.Mapped[User] (在 Post 中),　so.WriteOnlyMapped['Post'] (在 User 中)
對象數量,　單一 (One)　　　　　　　　　　,　多個 (Many)
讀取行為,　直接載入物件。　　　　　　　　,　回傳查詢物件，不預先載入資料。
適用場景,　當你確定關聯對象數量很少時。　,　當關聯對象可能非常龐大（如貼文、日誌）。
主要目的,　便利性。　　　　　　　　　　　,　效能與記憶體安全性。
